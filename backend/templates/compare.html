Parts B and C are truncated SHA-256 hashes: Part B captures which cipher suites you offered (order-independent), Part C captures which extensions and signature algorithms. Two clients with the same Part A but different Part B are running different TLS libraries. Same Part B but different Part C usually means the same library on a different OS or version.
